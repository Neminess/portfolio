This is a short documentation about OCR

OCR-what is it?

OCR comes from Optical Character Recognition. It is the technique used to scan documents into editable from, transfer information from paper to apps like excel, hand-held reading devices and virtual glasses meant to aid seeing-impaired people.

OCR-history

OCR was first developed in the 80s; the first device was made to make books more accessible to blind people.

The first OCR-devices didn't use a voice to read the letters it scanned. Instead, it let out beeps andd boops of different frequencies to the user via headphones.

    First devices:
  • 1870–1931 Fournier d'Albe's Optophone
  • 1931–1954 First OCR tools are invented and applied in industry, able to interpret Morse code and read text out loud
  • 1954–1974 The Optacon, the first portable OCR device, is developed
  • 1974–2000 Kurzweil Computer Products Inc develops the first omni-font OCR software, capable of reading any text document.
  • 2000–2016 OCR software is made available online for free, through products like Adobe Acrobat, WebOCR, and Google Drive
How it works

OCR works by detecting differences in hues in the text it scans. The technology detects the blacks and whites in an image, determining either pixel by pixel which part is black and white or by taking a group of pixels and determining the differences between them. The darker pixels are made black, lighter ones are white.

Modern usage

Today, OCR is used in banks to transfer checks to online-accounts, in VR-devices for the blind and seeing-impaired to read bottle labels and even detect objects (and where the objects are).

The technology is also used in cars that detect traffic-signs, in offices to transfer paper-documents to digitally editable form etc.

There are many applications and uses for OCR-technology, the main one in my opinion being the VR-glasses and devices for the seeing-impaired.

Conclusion

All in all, OCR is simple to program with the help of Tesseract OCR. It is the most popular and used module and for a good reason.

It works, and it works well. It is easy to use and best of all, it is free. So if one is going to make a program involving OCR, the Tesseract OCR is an exceptional option for the OCR-part of the code.

Tesseract is used with Python (other languages have their own import statements) by importing it with import pytesseract as tess. Then, the program is given a path to find the cmd for the tesseract-module/program.

Next, Image-is imported from PIL with a simple snippet: from PIL import Image. The image containing the text to be transferred into editable form is passed to a variable: var img=img.open(some_image) . After this the tesseract is called to action with

text=tess.image_to_string(img). Finally, the program or empty file where the text is destined to go is opened, the text is written with file.write(text) and you are done.
